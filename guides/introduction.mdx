The Partner Payments API is a server-to-server REST API for vetted partners — government agency case management systems, SaaS billing platforms, and similar integrations — to run sale, auth/capture, refund, void, and query operations directly against GroGov's payment infrastructure **without a citizen-facing browser flow**. Your backend calls our backend; citizens never leave your application.

## What you can do in v1

- **Create payments** — submit a `sale` (immediate capture) or an `auth` (hold funds, capture later).
- **Capture** — convert an authorized hold into a settled charge.
- **Refund** — full refund, partial refund, or refund against a specific line item.
- **Void** — cancel an authorized hold before capture.
- **Get a payment** — retrieve a single payment by transaction ID or line-item ID.
- **List payments** — filter by portal, status, date range, and more; results are cursor-paginated.

<Note>
**Out of scope in v1:** Webhooks are not available — poll the get/list endpoints to track status changes. ACH/e-check, Apple Pay, and Google Pay are not supported. Settlement, batch, and deposit endpoints are not exposed to partners; use the get/list endpoints to reconcile.
</Note>

## Conventions

**Response envelope.** Every response — success or failure — is wrapped in a consistent JSON envelope. A successful call returns:

```json
{ "success": true, "data": { ... } }
```

An error returns:

```json
{ "success": false, "message": "Human-readable description.", "details": { "code": "ERROR_CODE" } }
```

**Amounts are integer minor units (cents).** The `amount` field is always an integer representing cents: `100` = $1.00, `10000` = $100.00. There are no decimals anywhere in the API.

**Currency.** `currency` is always `USD` in v1. Include it in every request body.
