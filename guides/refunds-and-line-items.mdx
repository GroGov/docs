---
title: "Refunds & line items"
description: "Group vs. line-item refunds, partial amounts, and pro-rated fees."
---

Every payment is a **group** containing one or more **line items**. Refunds can target either the whole group or an individual line item, giving you fine-grained control over partial reversals.

## Group and line-item IDs

- The **group ID** is the top-level `transactionId` on the payment object.
- Each **line item** has its own `id`.

Both identifiers can be passed in the `{id}` path parameter of `POST /payments/{id}/refund` ã the API determines the scope automatically.

## Refund scope

| `{id}` value | Scope | Response type |
|---|---|---|
| Group `transactionId` | Refunds the whole group, or a partial `amount` | `RefundGroupResponse` ã includes `target: "group"` |
| Line-item `id` | Refunds that item only, capped at its `remainingRefundableAmount` | `RefundLineItemResponse` ã includes `target: "line_item"`, `lineItemRemainingRefundableAmount`, and `groupRemainingRefundableAmount` |

## `reversalType`

The `reversalType` field in the response tells you how the reversal was processed:

- `"void"` ã the batch was still open; the authorisation was cancelled before settlement.
- `"refund"` ã the batch had already closed; a refund transaction was issued.

This is chosen automatically ã you do not need to specify it.

## Service fees

Service fees are **pro-rated** on partial and line-item refunds. The refunded portion of the fee is surfaced as `refundedServiceFee` on the refund response. Per line item, the fee is allocated proportionally to that item's share of the total.

## Examples

### Full group refund

<CodeGroup>

```bash cURL
curl -X POST https://apistagepc.grogov.us/api/partner/v1/payments/a1b2c3d4-e5f6-7890-abcd-ef1234567890/refund \
  -H "Authorization: ApiKey pc_<your-key>" \
  -H "Content-Type: application/json" \
  -d '{ "reason": "Case dismissed" }'
```

```js Node.js
const res = await fetch(
  "https://apistagepc.grogov.us/api/partner/v1/payments/a1b2c3d4-e5f6-7890-abcd-ef1234567890/refund",
  {
    method: "POST",
    headers: {
      Authorization: "ApiKey pc_<your-key>",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ reason: "Case dismissed" }),
  }
);
const data = await res.json();
```

</CodeGroup>

Response:

```json
{
  "success": true,
  "data": {
    "transactionId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "target": "group",
    "reversalType": "refund",
    "amount": 10300,
    "currency": "USD",
    "reason": "Case dismissed",
    "status": "succeeded",
    "remainingRefundableAmount": 0,
    "gateway": { "transactionId": "4938271649" },
    "createdAt": "2024-03-01T12:00:00.000Z"
  }
}
```

### Line-item partial refund

<CodeGroup>

```bash cURL
curl -X POST https://apistagepc.grogov.us/api/partner/v1/payments/b2c3d4e5-f6a7-8901-bcde-f12345678901/refund \
  -H "Authorization: ApiKey pc_<your-key>" \
  -H "Content-Type: application/json" \
  -d '{ "amount": 5000, "reason": "Partial fee waiver" }'
```

```js Node.js
const res = await fetch(
  "https://apistagepc.grogov.us/api/partner/v1/payments/b2c3d4e5-f6a7-8901-bcde-f12345678901/refund",
  {
    method: "POST",
    headers: {
      Authorization: "ApiKey pc_<your-key>",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ amount: 5000, reason: "Partial fee waiver" }),
  }
);
const data = await res.json();
```

</CodeGroup>

Response:

```json
{
  "success": true,
  "data": {
    "transactionId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "target": "line_item",
    "targetId": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
    "reversalType": "void",
    "amount": 5000,
    "refundedServiceFee": 150,
    "currency": "USD",
    "status": "succeeded",
    "lineItemRemainingRefundableAmount": 0,
    "groupRemainingRefundableAmount": 5150,
    "gateway": { "transactionId": "4938271650" },
    "createdAt": "2024-03-01T12:00:00.000Z"
  }
}
```

## Error cases

<Warning>
Requesting a refund that exceeds the remaining refundable balance ã including refunding an already fully-refunded group ã returns `422 REFUND_EXCEEDS_AMOUNT`. See [Errors](/guides/errors) for the full code list.
</Warning>

## Idempotency

Refund requests support idempotency. Pass an `Idempotency-Key` header to make retries safe. See [Idempotency](/guides/idempotency) for details.
