## Magic card tokens

Pass these as `paymentMethod` in a create-payment request to trigger specific gateway behaviors deterministically.

| `paymentMethod` value | Result |
|---|---|
| `{ "type": "card_token", "token": "pc_test_card_approved" }` | Succeeds → `status: succeeded` |
| `{ "type": "card_token", "token": "pc_test_card_declined" }` | `402 PAYMENT_DECLINED` (soft decline) |
| `{ "type": "card_token", "token": "pc_test_card_timeout" }` | `504 PAYMENT_TIMEOUT` (gateway hang) |
| `{ "type": "card_token", "token": "pc_test_card_error" }` | `402 PAYMENT_DECLINED` (gateway 500 path) |
| `{ "type": "card_token", "token": "pc_test_card_malformed" }` | `402 PAYMENT_DECLINED` (malformed gateway 2xx) |
| `{ "type": "vault", "vaultId": "pc_test_vault_approved" }` | Succeeds (stored-card charge) |
| `{ "type": "vault", "vaultId": "pc_test_vault_declined" }` | `402 PAYMENT_DECLINED` |

## Worked scenarios

Each snippet uses `Authorization: ApiKey pc_…` directly so it is self-contained — no JWT exchange required. Replace `$(uuidgen)` with any unique string if your shell does not have `uuidgen`.

### 1. Successful sale (F-301 TC-01)

Charge `pc_test_card_approved` against portal A. Returns `status: "succeeded"` immediately.

```bash cURL
curl -s -X POST https://apistagepc.grogov.us/api/partner/v1/payments \
  -H "Authorization: ApiKey pc_test_alpha_key_9f2b7c1d4e8a" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{
    "portalId": "5a9b0004-0000-4000-a000-000000000004",
    "amount": 5000,
    "currency": "USD",
    "capture": "sale",
    "paymentMethod": { "type": "card_token", "token": "pc_test_card_approved" },
    "payer": { "firstName": "Jane", "lastName": "Doe", "email": "jane.doe@example.com" },
    "lineItems": [
      { "amount": 5000, "referenceId": "CASE-2024-001", "description": "Court filing fee" }
    ]
  }'
```

### 4. Timeout (F-301 TC-18)

### 7. Idempotent replay (F-306 TC-01)

Send the exact same request body and `Idempotency-Key` twice. The second response is identical to the first and carries the `Idempotent-Replayed: true` response header. Replace `my-idempotency-key-001` with any fixed string; the key must be the same in both calls.

```bash cURL
# First call — processes normally
curl -s -X POST https://apistagepc.grogov.us/api/partner/v1/payments \
  -H "Authorization: ApiKey pc_test_alpha_key_9f2b7c1d4e8a" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: my-idempotency-key-001" \
  -d '{
    "portalId": "5a9b0004-0000-4000-a000-000000000004",
    "amount": 5000,
    "currency": "USD",
    "capture": "sale",
    "paymentMethod": { "type": "card_token", "token": "pc_test_card_approved" },
    "payer": { "firstName": "Jane", "lastName": "Doe", "email": "jane.doe@example.com" },
    "lineItems": [
      { "amount": 5000, "referenceId": "CASE-2024-001", "description": "Court filing fee" }
    ]
  }'

# Second call — same key and body, returns cached response with Idempotent-Replayed: true
curl -si -X POST https://apistagepc.grogov.us/api/partner/v1/payments \
  -H "Authorization: ApiKey pc_test_alpha_key_9f2b7c1d4e8a" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: my-idempotency-key-001" \
  -d '{
    "portalId": "5a9b0004-0000-4000-a000-000000000004",
    "amount": 5000,
    "currency": "USD",
    "capture": "sale",
    "paymentMethod": { "type": "card_token", "token": "pc_test_card_approved" },
    "payer": { "firstName": "Jane", "lastName": "Doe", "email": "jane.doe@example.com" },
    "lineItems": [
      { "amount": 5000, "referenceId": "CASE-2024-001", "description": "Court filing fee" }
    ]
  }'
```

### 8. Line-item refund (F-303 TC-03)

Refund a single line item by its ID. The line-item ID goes in the URL path — use the `id` field from `lineItems[].id` in a prior create or get-payment response (not the group `transactionId`). The response includes `target: "line_item"` with updated `lineItemRemainingRefundableAmount` and `groupRemainingRefundableAmount`. Replace `$LINE_ITEM_ID` with the line-item `id`.

```bash cURL
curl -s -X POST https://apistagepc.grogov.us/api/partner/v1/payments/$LINE_ITEM_ID/refund \
  -H "Authorization: ApiKey pc_test_alpha_key_9f2b7c1d4e8a" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{ "amount": 5000 }'
```
